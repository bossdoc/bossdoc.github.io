Performance Module¶

Main idea for fitting – Get data as ship_wise_data from noon reports, VA or any other source. Filter it using relevant filtering parameters (laden, ballast, wind force etc.). Add stw, perf_speed if needed. Fit models.

Eg.

def test_noon_data_ship_wise():
        ship_list = models.Ship.objects.all().order_by("ship_name")[:100]
        ship_list = list(ship_list)
        filter_param = {
                "wind_force"                    :       [0,6],
                "condition"                             :       "All",
                "speed_range"                   :       [6, 16]
        }
        date_to = datetime.now()
        date_from = date_to - timedelta(days = 365)


        st = time.time()
        ship_wise_data = perf_data_analysis.get_noon_data_ship_wise(ship_list, filter_param, date_from, date_to)
        print("time to fetch data", time.time() - st)
        perf_data_analysis.add_stw_and_perf_speed(ship_wise_data)
        print("time to add stw/perf_speed data", time.time() - st)
        perf_data_analysis.fit_vessel_performance_models(ship_wise_data, single_order_fitting=False, fit_intercept=True)
        print("time to fit perf mdoels", time.time() - st)
        perf_data_analysis.fit_vessel_performance_models_windowed(ship_wise_data, windowing_type="monthly")
        print("time to fit windowed perf mdoels", time.time() - st)

vast.views_boss_capt.perf_data_analysis.get_compare_capt_data(ship_list=None, filter_param=None, start_date=None, end_date=None, ship_dry_dock_dates=None, use_speed_range=False, fetch_proj=False)¶

ship_list : List of ship objects filter_param : {

“wind_force” : [min,max], “condition” : # All, laden, ballast “speed_range” : [min, max]

} use_speed_range : Set true to use the speed range supplied in filter param set fetch_proj true to also fetch projected weather in noon reports. ship_drydock_dates is like :

ship_dates[ship.id] = {
“start_date”:date_from, “end_date”:date_to, “start_date_second”:None, “end_date_second”:None

}

if start_date_second, end_date_second is None only primary data is relevant. In practice, dont specify most filters in this function. Instead get all data, and use the filter_ship_wise_data function to filter data.

vast.views_boss_capt.perf_data_analysis.get_foc_from_noon(noon)¶

computes ME, AE, and total consumption of a record. If ‘foc’ key already present, does not do gradewise computation.

vast.views_boss_capt.perf_data_analysis.get_noon_data_ship_wise(ship_list, filter_params, date_from, date_to, hull_cleaning_dict=None)¶

ALWAYS use this function to get noon data for ships. ship_list : list of instances of models.Ship only relevant filter_params filter_params = {

“wind_force” : [min_wind_force,max_wind_force], “condition” : “All”,

} Use filter_ship_wise_data for more filtering. returns ship_wise_data : ship_wise_data : {

ship_id : [list of dicts, each dict is a noon]

}

vast.views_boss_capt.perf_data_analysis.get_voyage_abstract_data(ship_list, date_from=None, date_to=None, max_bft=5.0)¶

Use this function as a template to know what fields are required to do performance analysis and curve fittting. Datastructure similar to that returned by get_noon_data_ship_wise but fewer fields. enough for curve fitting.

vast.views_boss_capt.perf_data_analysis.window_data(ship_wise_data, windowing_type)¶

windowing_type : can be monthly, quarterly, yearly adds windowed data to ship_wise_data monthly: ship_wise_data[ship][‘windowed_data’] = {

‘2020_m_1’ : # jan 2020, ‘2020_m_2’ : # feb 2020,

} or quarterly ship_wise_data[ship][‘windowed_data’] = {

‘2020_q_1’ : # Q1 2020, ‘2020_q_2’ : # Q2 2020,

} or yearly ship_wise_data[ship][‘windowed_data’] = {

‘y_2020’ : # 2020, ‘y_2021’ : # 2021,

}
